# КОНТЕКСТ ПРОЕКТА: ПараПланнер (wed-control)

## 1. Суть проекта
Веб-сервис для свадебных организаторов (планирование, рассадка, тайминг).
Аудитория: Профи ивент-индустрии.
Мой проект лежит в папке wed-control. Я новичок. Всегда проверяй, нахожусь ли я в нужной папке. После любых изменений кода всегда пиши отдельным блоком, какую команду ввести в терминале, чтобы применить изменения и проверить, что всё работает.

## 2. Технический стек (СТРОГО СОБЛЮДАТЬ)
* **Фреймворк:** React (Vite)
* **Язык:** JavaScript (JSX)
* **Стили:** Tailwind CSS (классы внутри JSX)
* **Бэкенд:** Google Firebase (Firestore + Auth)
* **Деплой:** Vercel

## 3. Глобальные правила кодинга
1.  **Не ломай старое:** Новые функции не должны ломать существующий функционал.
2.  **Полный код:** Выдавай код изменяемого файла ЦЕЛИКОМ.
3.  **Терминал:** Команды пиши отдельным блоком.
4.  **Стили:** Используй только Tailwind CSS.

## 4. БЕЗОПАСНОСТЬ ДАННЫХ (КРИТИЧНО)
* **Firebase:** Данные хранятся в Firestore.
* **Запрет на переименование:** НИКОГДА не меняй названия полей, которые ссылаются на базу (например, client_id -> clientId). Это приведет к потере данных у клиентов.
* **Структура:** Старые данные должны корректно отображаться.
